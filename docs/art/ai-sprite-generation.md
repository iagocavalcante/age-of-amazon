# AI Sprite Generation — Prompts & Pipeline

> **Status:** spec validated end-to-end with OpenAI `gpt-image-1`
> (see "Validation log"). Walk-frame chaining still untested.
> **Goal:** richer pixel-art sprites with real per-tribe shape identity,
> generated by an image model, post-processed into clean game-ready PNGs.
> This document is self-contained on purpose: any future session (or a human
> with any image generator) can execute it without other context.

## Why this exists

All current art is procedural (see ADR in `docs/architecture.md`). The four
tribes differ only by palette tint. This pipeline produces *painted* sprites
with distinct silhouettes per tribe, imported as PNGs with a procedural
fallback so the game never breaks if an asset is missing.

## Art direction bible (constraints every prompt must respect)

- **View:** strict side view, character facing RIGHT. The engine flips
  horizontally for left. Never three-quarter view, never front-facing.
- **Target sizes (after downscale):** units 20×30 px, animals ≤ 40×24 px,
  1×1 buildings ~56×64 px, 2×2 buildings ~120×110 px. Generate large
  (1024×1024), downscale later — models cannot draw true low-res.
- **Anchor palette (the game's existing colors — keep generated art in this
  family):** outline `#18120E`, skin `#C49064`, skin shadow `#A4744E`,
  wood `#7A5630`, thatch `#98803E`, leaf `#547A3A`, stone `#A08C6C`,
  jade `#389E7E`, gold `#E4B45C`. Grass background in game is `#76AC3E`
  family — sprites must contrast with it.
- **Tribe accent = PURE MAGENTA `#FF00FF`.** Any clothing/feather/banner
  area that should take the player's color must be painted flat magenta;
  the pipeline remaps it per tribe (blue/red/yellow/purple). Never use
  magenta for anything else.
- **Background:** flat solid `#00FF00` green-screen, no shadows on the
  ground (the engine draws its own), no vignette, no border.
- **Style anchors:** "16-bit era strategy game sprite", "Age of Empires II
  unit style", hard black-brown outline, flat cel shading with at most 3
  shades per material, no anti-aliasing, no gradients, chunky readable
  forms. Readability at 200% zoom of final size beats detail.

## Master style prompt (prefix for EVERY generation)

```
Pixel art sprite for a 16-bit isometric RTS game set in the Amazon
rainforest, in the style of Age of Empires II unit graphics. Strict side
view facing right, full body, centered, large chunky pixels, hard dark
brown outline (#18120E), flat cel shading with maximum 3 shades per
material, NO anti-aliasing, NO gradients, NO ground shadow. Flat solid
#00FF00 background. Clothing and feather accents painted in flat pure
magenta #FF00FF (they will be recolored per player). Earthy palette:
skin #C49064, wood #7A5630, thatch #98803E, leaf green #547A3A.
The design must be recognizable as a tiny 16x24 pixel sprite.
```

## Per-asset prompts (append to the master prompt)

### Units — generate one image per FRAME, chaining for consistency
Frame chaining: generate the IDLE frame first, then use image-editing mode
with the idle image as input and the prompt "Exact same character and style,
now mid-stride with the LEFT leg forward" (then RIGHT) for walk frames.
Never generate walk frames from scratch — they will drift.

- **Villager (per tribe N):** `A tribal villager of tribe N: [tribe motif
  from the table]. Simple magenta tunic, bare feet, carrying nothing.
  Standing relaxed, arms at sides.`
- **Warrior (per tribe N):** `A tribal warrior of tribe N: [motif]. Magenta
  chest wrap, wooden spear held vertically in right hand, small round
  hide shield on left arm, stone spear tip.`
- **Archer (per tribe N):** `A tribal archer of tribe N: [motif]. Magenta
  sash, wooden longbow held in left hand at side, quiver of arrows on
  back, slighter build than the warrior.`

### Buildings — one image each, static
- **Town Center (per tribe):** `A stepped rainforest pyramid of packed
  earth and stone, 3 tiers, central stairway, doorway at base, two magenta
  banner poles. Tribe N trim: [motif applied to architecture].`
- **House:** `A small rainforest hut: timber plank walls, layered palm
  thatch roof, doorway with a magenta lintel banner.`
- **Barracks:** `A wide tribal longhouse: ridged thatch roof, timber walls,
  a rack of spears beside the door, a magenta round shield on the wall.`
- **Watchtower:** `A tall lookout tower on four wooden stilts with cross
  bracing, railed platform, small thatch cap roof, long magenta pennant.`
- **Monument:** `A tall carved jade monolith on a two-step stone plinth,
  glowing green jade with gold inlay bands and carved glyphs, jade
  capstone, magenta pennant. Sacred, imposing.`

### Environment trees (neutral — NO magenta anywhere)
Three real Amazon species, sized so the forest towers over the 20x30 unit
(kapok 56x72, brazil nut 44x64, acai 28x52). Prompt with "full tree,
centered, ground-level base"; for slender species force a "tall vertical
composition" or the model renders a squat clump (bit the first acai).
- **Kapok (samauma):** giant pale trunk, wide buttress roots, high flat
  umbrella crown.
- **Brazil nut (castanheira):** very tall bare cylindrical trunk, single
  dense rounded crown at the top.
- **Acai palm:** two slender ringed stems, feathery frond bursts, dark
  purple berry clusters under the crowns.

### Wildlife (neutral — NO magenta anywhere)
- **Capybara:** `A capybara, brown, rounded, calm.`
- **Jaguar:** `A jaguar, golden coat with dark rosettes, prowling.`
- **Tapir:** `A lowland tapir, dark grey-brown, stocky, short trunk.`
- **Bush dog:** `A bush dog, small rusty-brown, short legs, alert.`
- **Black caiman:** `A black caiman, long low armored body, olive-dark
  scutes, seen from the side as if at a river edge.`

## Post-processing pipeline (run per generated image)

```python
# python3, pillow: pip install pillow
from PIL import Image

SRC = "raw.png"; OUT = "sprite.png"
TARGET = (16, 24)              # per asset type, see bible
img = Image.open(SRC).convert("RGB")

# 1. Key out the green screen -> alpha
img = img.convert("RGBA")
px = img.load()
for y in range(img.height):
    for x in range(img.width):
        r, g, b, a = px[x, y]
        if g > 180 and r < 120 and b < 120:
            px[x, y] = (0, 0, 0, 0)

# 2. Crop to content bounding box, then downscale NEAREST
img = img.crop(img.getbbox())
img = img.resize(TARGET, Image.NEAREST)

# 3. Quantize to a tight palette (keeps the pixel-art crunch)
alpha = img.getchannel("A")
img = img.convert("RGB").quantize(colors=14, dither=Image.NONE).convert("RGBA")
img.putalpha(alpha.point(lambda a: 255 if a > 96 else 0))

img.save(OUT)
```

Magenta→tribe tint happens at LOAD time in the engine (see below), not in
the pipeline, so one file serves all four tribes when the per-tribe motifs
are skipped, or four files when they are not.

## Integration plan (engine side)

1. Assets live in `assets/sprites/<name>.png`
   (e.g. `unit_warrior_idle_t1.png`, `building_house.png`).
2. `AssetLibrary` gains a loader that PREFERS a PNG when it exists and
   falls back to the current procedural art otherwise — the game must
   never break on a missing file, and the procedural system remains the
   canonical baseline (keeps the ADR honest).
3. Tinting: at load, every pixel with `r>200, b>200, g<100` is remapped to
   `Constants.PLAYER_COLORS[player_id]` (preserving its value/shade), the
   same role magenta plays in the prompts.
4. Update `docs/architecture.md` with an ADR when the first painted asset
   ships: "hybrid art — procedural baseline, optional painted overrides".

## Validation checklist (per sprite, before committing)

- [ ] Reads at actual in-game size against grass `#76AC3E` (paste on swatch)
- [ ] Silhouette identifies the unit type AND tribe with color removed
- [ ] Walk frames align (feet baseline within 1px across frames)
- [ ] No stray magenta/green-screen pixels after processing
- [ ] File under `assets/sprites/`, snake_case name from the table above

## Validation log

  draws the green screen as an inset box on a different outer background

  fill: any pixel with `g > 140 and g > 1.5*r and g > 1.5*b` is keyed.
