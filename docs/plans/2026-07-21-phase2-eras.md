# Phase 2 — The Three Eras Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add an Age-of-Empires-style **three-era progression** (Forest → Village
→ Chiefdom) that gates units, buildings, and upgrades; costs resources to
advance; applies a small tribe-wide buff each age; is driven by the AI on the
same rules; and replicates + saves. Plus the Phase-2 **land content**: Hunter,
Storehouse, Palisade (wall + gate), and Shaman. The **War Canoe is explicitly
DEFERRED** to Phase 2.5 (it needs water-borne movement, a new domain).

**Architecture:** Era state is **per-player** (each tribe advances
independently), held in `GameManager` next to stockpiles, mutated only through
an authoritative `advance_era` command in `CommandRouter`, and read by a single
`Constants`-driven gate (`def["era"]`) that both the placement/training
validators and the HUD consult. Era state replicates like stockpiles and
serializes like them. New units/buildings are **data entries** in `UNIT_DEFS`/
`BUILDING_DEFS` plus, where a genuinely new behaviour is needed (heal aura,
drop-off, wall), a small extension to the one system that owns it. Everything
stays symmetric for the AI (design pillar P2) and inside the web build (P4).

**Tech Stack:** Godot 4.5, GDScript. Verification via headless `--test-*`
harnesses (ADR 12); the multiplayer harness `tools/test_mp.sh`; visual capture
in movie-writer mode for UI.

`GameManager.gd`:
```gdscript
# Can this player advance to the next era right now (requirements met)?
# Does NOT check cost — cost is checked+charged atomically in advance_era.
func can_advance_era(player_id: int) -> bool:
	var next: int = player_era(player_id) + 1
	if not Constants.ERA_DEFS.has(next):
		return false
	var need: Dictionary = {}
	for bt: String in Constants.ERA_DEFS[next]["requires_buildings"]:
		need[bt] = need.get(bt, 0) + 1
	var have: Dictionary = {}
	for node: Node in get_tree().get_nodes_in_group("player_%d" % player_id):
		var b: Building = node as Building
		if b != null and b.is_constructed:
			have[b.building_type] = have.get(b.building_type, 0) + 1
	for bt: String in need:
		if have.get(bt, 0) < need[bt]:
			return false
	return true

# Authoritative advance: checks requirements + cost, charges, bumps era, emits.
# Returns true on success. Callers must be the authority (CommandRouter).
func advance_era(player_id: int) -> bool:
	var next: int = player_era(player_id) + 1
	if not Constants.ERA_DEFS.has(next):
		return false
	if not can_advance_era(player_id):
		return false
	var cost: Dictionary = Constants.ERA_DEFS[next]["advance_cost"]
	if not can_afford(player_id, cost):
		return false
	spend(player_id, cost)
	eras[player_id] = next
	EventBus.era_advanced.emit(player_id, next)
	_replicate_era(player_id)   # implemented in A4; stub as no-op until then
	return true
```
(For A2, `_replicate_era` can be a one-line empty method; A4 fills it.)
